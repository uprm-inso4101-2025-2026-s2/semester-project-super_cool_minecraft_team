In the risk tree above, we represent the different exceptions and risks within our application that could cause a timeout or security risk. These risks have already been mitigated, since we have 6 different test files in our repository (DependencyControllerTest.java, MainControllerTest.java, MinecraftProjectApplicationTest.java, RedirectControllerTest.java,GraphServiceTest.java,GraphDSTests.java) we created after identifying these risks. Once the tests were created, we implemented exception handling and better security for each of these risks, and ran the tests. All have passed or served their purpose. The tests are designed to stop those specific failures from ever traveling up through the logical gates to cause the top-level system failure.

At the absolute top sits the catastrophic failure state: Mod Tool Application Deployment or Production Failure.

The Intermediate Drivers (Rectangles): The root failure is broken down into three main operational threat domains: a Security Boundary Breach, a Runtime/Startup Context Crash, and Core Feature Performance Degradation.

The Root Causes (Ovals): At the very bottom of the branches are the leaf nodes, representing the specific technical errors that trigger the risks, such as unhandled network timeouts, malicious phishing inputs, or circular dependency injections. Connecting the nodes are OR gates. This logical setup dictates that the parent event will be triggered if any single child failure occurs. For instance, the system does not require both a rate-limit crash and a network timeout to break the application API; an outage or exception in either one independently will pass right through the gate and compromise the core feature. The green shield labels underneath the ovals show where our test files interact with the risk tree. By checking for bad inputs, mocking server timeouts, or validating bean injections, these test files act as defensive line-guards that catch and eliminate the root failure states before they can move up the logical gates to crash our deployment.
